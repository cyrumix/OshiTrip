# ADR-0005: ローカル永続化 / オフラインキャッシュに Drift を採用

- 状態: 承認済み
- 日付: 2026-07-01
- 関連要件: §6.2, §15.3

## 背景 / 課題

当日重要情報のオフライン閲覧（§6.2）、ホームのキャッシュ先行表示、自動保存と通信復旧後の同期（§15.3）、二重送信防止のためのOutboxが必要。

## 選択肢

1. **Drift（SQLite）** — 型安全なクエリ、マイグレーション、リレーショナルキャッシュとOutboxに適合。テスト容易。
2. Isar / Hive — 高速だがリレーショナル表現とマイグレーションが弱い。
3. shared_preferences のみ — 構造化データ/一覧キャッシュに不足。

## 決定

**Drift（SQLite）** を採用。当日情報キャッシュ、一覧キャッシュ、Outbox（未同期変更キュー）をローカルDBで管理する。

## 影響

- 変更は「ローカルコミット→UI即時反映→Outbox→サーバー同期」。`client_mutation_id` で冪等化し二重送信を防止（§15.3）。
- チケット画像は Storage 署名URL経由でバイト列を認可付きキャッシュに保存し、ログには出さない（§15.2）。
- スキーマ/マイグレーションはサーバー（Supabase）と別に、端末ローカル用として管理し整合を取る。

## 検証方法

- 単体テストでOutboxの冪等再送・競合解決を検証。
- 統合テストでオフライン編集→接続回復→同期を検証。
