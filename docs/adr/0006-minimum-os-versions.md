# ADR-0006: サポート対象 OS 最低バージョン

- 状態: 承認済み
- 日付: 2026-07-01
- 関連要件: §15.1（「OSのサポート対象バージョンは実装開始時に確定する」）

## 背景 / 課題

Flutter安定版、主要依存（supabase_flutter、firebase_messaging、drift、image_picker、flutter_local_notifications）、ストア要件を踏まえ、最低OSを確定する。

## 選択肢

- iOS: 12 / 13 / 15 / 16
- Android: API 21(5.0) / 23(6.0) / 24(7.0) / 26(8.0)

考慮点:
- Firebase系プラグインは近年 iOS 13+ を要求する傾向。
- 通知チャンネル（§11 種類別設定）は Android **API 26+** でネイティブ対応。
- App Store提出はXcodeのSDK/最低iOS制約に従う。

## 決定

- **iOS 最低: 15.0**
- **Android 最低: API 26（Android 8.0）**、target/compile は導入時点の最新安定（現時点 API 35 を想定、SDK導入時に確定）

理由:
- iOS 15 は主要プラグインの動作が安定し、対象ユーザー（現行スマホ利用者）をほぼ網羅。
- Android 8.0 により**通知チャンネル**（§11 種類別の有効/無効・タイミング設定）と近代的な暗号化APIをネイティブに利用でき、要件の通知UXを素直に実装できる。カバレッジ上の損失は現行市場で軽微。

## 影響

- `ios/Podfile` の `platform :ios, '15.0'`、`android/app/build.gradle` の `minSdk 26`（SDK導入時に設定）。
- 実際の対応可否はプラグインバージョン確定後（ツールチェーン導入後）に再検証し、必要なら本ADRを改訂する。

## 検証方法

- CIビルドが両OSの最低バージョン設定で成功すること（ツールチェーン導入後）。
- 依存の最低OS要求がこれを上回らないことをpub導入時に確認。
