# ADR-0004: ルーティングに go_router を採用

- 状態: 承認済み
- 日付: 2026-07-01
- 関連要件: §4.1, §5, §8.3, §11

## 背景 / 課題

5タブのグローバルメニュー、認証/チュートリアルによるリダイレクト、通知タップからの対象画面ディープリンクが必要。

## 選択肢

1. **go_router** — 宣言的ルート、`redirect`、`StatefulShellRoute`（タブ状態保持）、ディープリンク対応。公式推奨。
2. auto_route — コード生成ベース。高機能だが導入コスト。
3. Navigator 2.0 手書き — 柔軟だが実装/保守コスト高。

## 決定

**go_router** を採用。

- `redirect` で「未認証→ログイン」「チュートリアル未完了→オンボーディング（§4.1 完了状態保存）」。
- `StatefulShellRoute` で ホーム/現場/思い出/マイ推し/設定（§5）の状態保持。
- 名前付きルート＋パラメータで通知/ディープリンク（§8.3, §11）を対象入力画面へ直接遷移。

## 影響

- ルート定義は `lib/app/router.dart` に集約。ディープリンクのパスはPhaseごとに追加。

## 検証方法

- Widget/統合テストでリダイレクト（未認証/チュートリアル）とディープリンク遷移を検証。
