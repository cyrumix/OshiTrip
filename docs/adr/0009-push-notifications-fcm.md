# ADR-0009: Push通知に FCM（+ APNs）を採用

- 状態: 承認済み
- 日付: 2026-07-01
- 関連要件: §4.1, §11, §16.1

## 背景 / 課題

要件 §16.1 が「Push通知: Firebase Cloud Messaging、iOSはAPNs連携」を明記。通知は用途説明→許可要求の順（§11, §4.1）で、種類別に有効/無効とタイミングを設定可能にする。

## 決定

**FCM** を採用し、iOSは **APNs** と連携する（バックエンドのSupabaseとは独立）。

- クライアント: `firebase_messaging` でトークン取得/更新、`flutter_local_notifications` でフォアグラウンド表示と**通知チャンネル**（種類別設定, §11 / Android API 26+）。
- 許可要求は**チュートリアル中に行わず**、通知価値が生じた時点で説明後に要求（§4.1, §11）。
- 送信スケジューリング（現場接近/Todo期限/未登録交通宿泊/思い出入力/記念日/共有更新, §11）はサーバー側（Supabase Edge Functions + スケジュール、またはcron）で判定し、完了済み/不要指定済みには送らない。同日通知はまとめる（§11）。
- ディープリンク: 通知payloadに対象ルートを含め、タップで対象入力画面へ直接遷移（§8.3, §11, ADR-0004）。

## 影響

- 環境別のFirebaseプロジェクト（ADR-0007）。`google-services.json` / `GoogleService-Info.plist` はgitignore（[.gitignore](../../.gitignore)）。
- iOSは APNs 認証キーの設定が必要（リリース工程, §16.2）。

## 検証方法

- 通知設定の有効/無効・タイミングがユーザー設定で反映されること（単体/Widgetテスト）。
- 「完了済み・不要指定済み項目に通知しない」ロジックの単体テスト（§11, §7.4/§7.5 の要否区別）。
- ディープリンク遷移の統合テスト。
